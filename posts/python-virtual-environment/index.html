Setting Up Python Virtual Environments in Ubuntu

Feb 1, 2018

NOTE: This guide will assume you are using Python 3.3 and up. We will be using the newer venv module in place of virtualenv.

Virtual environments help us avoid dependency issues that may arise in our Python projects. Imagine starting a project, and using v1.5 of a package in the project. At a later time, you update the same package to v.2 for another project. It is entirely possible that some or all of the code in your original project is now broken! Imagine using dozens of packages with your project and having this problem with all of them. To avoid this, virtual environments allow us to use both particular versions of Python as well as particular versions of packages in self-contained projects. Polluting your global Python environment is not a good idea, especially since your Linux system itself is so heavily reliant on its Python installation.

To set up a virtual environment, first ensure that you have the python3-venv package installed.

a=python3-venv; if apt list | grep -qw $a; then echo $a is installed; else sudo apt install $a; fi

Next, cd into your projects directory and enter the following command:

python3 -m venv my_new_virtual_env

The new virtual environment will have several directories and scripts inside. my_new_virtual_env/bin will contain a soft link to a Python binary or binaries, as well as activate scripts. To activate a virtual environment, simply enter:

source my_new_virtual_env/bin/activate

Now your system's Python executable will only act within your virtual environment. You can install and use packages as usual.

pip install project

Any installed packages will live in my_new_virtual_env/lib. To exit the virtual environment and return to the global environment:

deactivate
